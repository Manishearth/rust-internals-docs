Unstable (rustc_private)

A pass that removes various redundancies in the CFG. It should be called after every significant CFG modification to tidy things up.

This pass must also be run before any analysis passes because it removes dead blocks, and some of these can be ill-typed.

The cause of that is that typeck lets most blocks whose end is not reachable have an arbitrary return type, rather than having the usual () return type (as a note, typeck's notion of reachability is in fact slightly weaker than MIR CFG reachability - see #31617).

A standard example of the situation is: rust fn example() { let _a: char = { return; }; }

Here the block ({ return; }) has the return type char, rather than (), but the MIR we naively generate still contains the _a = () write in the unreachable block "after" the return.
